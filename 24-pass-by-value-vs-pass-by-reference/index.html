Pass by Value vs Pass by Reference in JavaScript

Pass by Value (Primitive Data Types)

Primitive data types (like numbers, strings, booleans) are passed by value. Changes to the copied value do not affect the original.


let a = 10;
let b = a; // b gets a copy of a's value
b = 20;    // Changing b does not affect a

Pass by Reference (Objects, Arrays, Functions)

Objects, arrays, and functions are passed by reference. Changes to the copied reference affect the original object.


let obj1 = { name: "Alice" };
let obj2 = obj1; // obj2 references the same object as obj1
obj2.name = "Bob"; // Changing obj2 affects obj1

Comparison by Reference

When comparing objects or arrays, JavaScript compares their memory addresses, not their values.


let obj1 = { name: "Alice" };
let obj2 = { name: "Alice" };
console.log(obj1 === obj2); // false (different memory addresses)
